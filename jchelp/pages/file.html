File

File Sub Menu.

New
Deletes the current model and starts a new one.

Store As.
Stores the current model within the Workshop App. Storage of models is temporary and are only kept while the app is open. A more permanent method is to save or export them.
However it is the simplest way to keep a range of models for quickly building up a scene.

Fetch.
Fetches a model from the Store. This will delete the current model.

Save As.
Saves the current model to the browsers local storage on your local computer. What is saved by one browser cannot be accessed by another. Using browser tools to clear the Cache can clear the local storage.
The only way to access the saved models is through the app that saved them.

Open.
Open a model from the local storage. This will delete the current model.

Manage.
Produces a list of saved models from the local storage. Individual models can then be deleted.

Export.
Opens a new browser window where the data describing the model is written. Using browser tools save this data to a file on the local computer. The saved file can be a txt or a babylon file.
The file may be used in your own babylon.js projects. If you use textures then you need to obtain copies of the appropriate texture images.

Import.
Browse to an exported model file and import into the construction room. This will delete the current model.
